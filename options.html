The Great Suspender :: Settings



Suspended Tab History

A history of the tabs that you have suspended in the past is recorded. In the (unlikely :) event that The Great Suspender should ever crash, you can recover lost tabs from the history page.

Notes about Screen Capturing:

The screen capturing is done using Niklas von Hertzen's most excellent html2canvas script which attempts to render the dom onto an html5 canvas element. As far as I understand, this is still a work in progress and will not always be able to accurately represent the contents of the page in an image, sometimes resulting in missing sections or even no image at all.

Please also note that turning on screen capturing will increase the memory footprint of the suspended page (somewhat contrary to the primary goal of this extension). Likewise, the larger the height of the preview image, the larger the memory footprint of the suspended page.

Notes about The Great Suspender:

The great suspender works by first calling chrome://kill on the tab to release all the memory that the tab might be holding on to. It then creates a simple html page consisting of a the url, title and favicon of the page being suspended as well as (optionally) a screen capture of a small section of the page being suspended. Clicking anywhere on this page will result in the navigation back to the previous page that was suspended. On a chrome restart all suspended pages will revert back to the live version.

Please also note that there is another chrome extension called Jolly Roger which takes advantage of chrome's experimental processes api to achieve a similar result to this extension but without having to resort to chrome://kill to unallocate memory. I have written this plugin as an alternative to Jolly Roger that doesn't rely on using api's that are still in the development stage.
